Operating mode

1 UPS devices (SNMP)

Monitor one or more UPS devices via SNMP. For servers with redundant power supplies use one UPS per feed — the mapping is done in step 2. Optional per-UPS thresholds override the global values from step 3.

2 Proxmox hosts

API token with the Sys.PowerMgmt privilege. The host marked ★ “This host” (the appliance) is shut down last.

3 Triggers (thresholds)

One matching condition is enough. Empty field = condition disabled.

Empty by default (fail-safe): an SNMP loss triggers no shutdown, because it is usually a network/UPS defect and not a power outage. Only set this when a prolonged communication loss should deliberately be treated like an outage — the actual power state is unknown then.
